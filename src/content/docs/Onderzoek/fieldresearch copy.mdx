Deze procedure helpt je bij het systematisch controleren van websites voordat ze live gaan. Bij salve hanteren we strikte kwaliteitsstandaarden.

### Pre-Launch Checklist

<Steps>

1. **Technische Controles**

   Controleer de volgende technische aspecten:

   - [ ] Alle pagina's laden correct (geen 404 errors)
   - [ ] Console errors zijn opgelost
   - [ ] Mobile responsiveness werkt op alle breakpoints
   - [ ] Cross-browser compatibiliteit (Chrome, Firefox, Safari, Edge)

2. **Performance Optimalisatie**

   Meet en optimaliseer de performance:

   - [ ] PageSpeed Insights score > 90 (desktop)
   - [ ] PageSpeed Insights score > 80 (mobile)
   - [ ] Afbeeldingen zijn geoptimaliseerd (WebP formaat waar mogelijk)
   - [ ] Lazy loading is geïmplementeerd

3. **SEO & Toegankelijkheid**

   Zorg voor goede vindbaarheid en toegankelijkheid:

   - [ ] Meta titles en descriptions zijn ingevuld
   - [ ] Alt-teksten voor alle afbeeldingen
   - [ ] Heading structuur is logisch (H1 → H2 → H3)
   - [ ] Observatory score > 80

4. **Content Verificatie**

   Controleer alle content:

   - [ ] Spelfouten en grammatica
   - [ ] Contactgegevens zijn correct
   - [ ] Links werken (interne en externe)
   - [ ] Formulieren functioneren correct

</Steps>

## Tools voor Kwaliteitscontrole

<Tabs>
<TabItem label="Performance">

### PageSpeed Insights

- **URL**: https://pagespeed.web.dev/
- **Doel**: Score > 90 (desktop), > 80 (mobile)
- **Let op**: Core Web Vitals (LCP, FID, CLS)

### GTmetrix

- **URL**: https://gtmetrix.com/
- **Doel**: Grade A, laadtijd < 3 seconden
- **Gebruik**: Gedetailleerde waterfall analyse

</TabItem>
<TabItem label="Toegankelijkheid">

### Mozilla Observatory

- **URL**: https://observatory.mozilla.org/
- **Doel**: Score > 80
- **Controleert**: Security headers, SSL, content security

### WAVE Web Accessibility

- **URL**: https://wave.webaim.org/
- **Doel**: 0 errors, minimale warnings
- **Controleert**: WCAG compliance

</TabItem>
<TabItem label="SEO">

### Google Search Console

- **Setup**: Voeg property toe voor nieuwe website
- **Controle**: Indexering, crawl errors, mobile usability

### Screaming Frog

- **Download**: https://www.screamingfrog.co.uk/
- **Gebruik**: Crawl website voor technische SEO issues

</TabItem>
</Tabs>

## Deployment Verificatie

Na deployment controleer je:

:::tip[Pro Tip]
Gebruik altijd een incognito/private browsing venster om caching issues te voorkomen.
:::

### Productie Checklist

- [ ] SSL certificaat is actief (groene slot in browser)
- [ ] Redirects werken correct (www → non-www of omgekeerd)
- [ ] Google Analytics/Tag Manager tracking werkt
- [ ] Contact formulieren versturen emails
- [ ] 404 pagina toont custom error page

### Monitoring Setup

- [ ] Google Analytics property aangemaakt
- [ ] Search Console property toegevoegd
- [ ] Uptime monitoring geconfigureerd (bijv. UptimeRobot)
- [ ] Backup systeem gecontroleerd

## Veelvoorkomende Issues

:::caution[Let Op]
Deze problemen komen vaak voor bij nieuwe websites:
:::

1. **Mixed Content Warnings**: HTTP resources op HTTPS pagina's
2. **Ontbrekende Favicon**: Zorgt voor console errors
3. **Grote Afbeeldingen**: Niet geoptimaliseerd voor web
4. **Ontbrekende Alt-teksten**: Slecht voor toegankelijkheid
5. **Broken Links**: Vooral naar externe websites
